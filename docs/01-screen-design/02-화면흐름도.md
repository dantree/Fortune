## 2. 화면 전환 상태도 (State Machine)

```mermaid
stateDiagram-v2
    [*] --> Landing: 최초 진입 / 다시하기

    Landing --> Validating: CTA 클릭
    Validating --> Landing: 검증 실패 (inline error)
    Validating --> Loading: 검증 성공 + sessionStorage 저장

    Loading --> Result: timer 2500ms + 계산 완료

    Result --> Landing: "다시 하기" (clearSession=true)
    Result --> Landing: "다시 하기" (clearSession=false, 값 유지)
    Result --> SharePreview: "이미지 저장" (선택)

    Loading --> Landing: browser back

    SharePreview --> Result: 닫기

    note right of Loading
        AdSlot: ads-loading
        Interstitial 가능
    end note

    note right of Result
        AdSlot: ads-result
        Anchor Ad 전역
    end note
```

## 3. 화면별 이벤트 흐름

### 3.1 SCR-00 → SCR-01

```mermaid
sequenceDiagram
    actor U as 사용자
    participant UI as 입력 화면
    participant V as Validator
    participant S as SessionStore
    participant R as Router

    U->>UI: 생년월일·MBTI 입력
    UI->>UI: birthDate 변경 → 오행 배지 갱신
    U->>UI: "결과 보기" 클릭
    UI->>V: validate(formData)
    alt invalid
        V-->>UI: errors[]
        UI-->>U: inline error 표시
    else valid
        V->>S: setItem('fortune_input', JSON)
        V->>R: navigate('#/loading')
        R-->>UI: SCR-01 렌더
    end
```

### 3.2 SCR-01 → SCR-02

```mermaid
sequenceDiagram
    actor U as 사용자
    participant L as 로딩 화면
    participant E as Engine
    participant A as AdSlot
    participant R as Router

    L->>A: ads-loading mount
    L->>L: progress 0→100% (2.5s)
    par 계산
        L->>E: calculate(input)
        E-->>L: result JSON
    and UI
        L->>L: 메시지 순환 (3단계)
    end
    L->>S: setItem('fortune_result', JSON)
    L->>R: navigate('#/result')
    R-->>U: SCR-02 + 점수 표시
```

## 5. 에러·예외 흐름

| 상황 | 처리 | 화면 |
|------|------|------|
| 생년월일 미래 날짜 | inline error | SCR-00 |
| MBTI 미선택 | 필드 highlight | SCR-00 |
| sessionStorage 없이 `#/result` 직접 접근 | `#/` redirect | SCR-00 |
| sessionStorage 없이 `#/loading` 직접 접근 | `#/` redirect | SCR-00 |
| AdSense 로드 실패 | 슬롯 collapse (min-height 0) | 전체 |
| JS 비활성 | `<noscript>` 안내 | SCR-00 |

## 7. 화면 ID ↔ Hash 라우팅 매핑

| Hash | 화면 ID | 진입 조건 |
|------|---------|----------|
| `#/` 또는 `` | SCR-00 | 기본 |
| `#/loading` | SCR-01 | 유효한 입력 + sessionStorage |
| `#/result` | SCR-02 | 계산 결과 sessionStorage |
| `#/result?share=1` | SCR-03 | 결과 + 캡처 모드 |
