## 2. 화면별 상세 정의

### SCR-00 — 랜딩(입력) 화면

#### 2.1 목적
- SNS·블로그 유입 사용자가 **30초 이내**에 테스트를 시작할 수 있게 한다.
- 두 사람(본인·상대)의 **생년월일 + MBTI**를 수집한다.

#### 2.2 레이아웃 구역

| 구역 ID | 명칭 | 높이(모바일) | 내용 |
|---------|------|-------------|------|
| AD-TOP | 상단 광고 | 90~120px | 애드센스 디스플레이 (선택, 첫 방문 시 비활성 가능) |
| HDR | 헤더 | 56px | 로고·서비스명 "MBTI × 오행 궁합" |
| HERO | 히어로 | 120px | 한 줄 카피 + 이모지 |
| FORM-A | 본인 입력 | 가변 | 생년월일, MBTI |
| FORM-B | 상대 입력 | 가변 | 생년월일, MBTI |
| CTA | 결과 보기 버튼 | 52px | Primary CTA |
| FTR | 푸터 | 80px | kaltaelee.com 블로그 링크, 개인정보 안내 |

#### 2.3 입력 필드

| 필드 ID | 라벨 | 타입 | 검증 | 비고 |
|---------|------|------|------|------|
| `personA.birthDate` | 나의 생년월일 | date | 1900-01-01 ~ 오늘 | 필수 |
| `personA.mbti` | 나의 MBTI | select/modal | 16유형 중 1 | 필수 |
| `personB.birthDate` | 상대 생년월일 | date | 동일 | 필수 |
| `personB.mbti` | 상대 MBTI | select/modal | 16유형 중 1 | 필수 |

#### 2.4 오행 자동 표시 (읽기 전용)
- 생년월일 입력 완료 시 해당 인물 카드 하단에 **일간 기준 오행** 배지 표시  
  예: `🌳 목(木)` / `🔥 화(火)` / `🌍 토(土)` / `⚙️ 금(金)` / `💧 수(水)`
- MBTI 선택 시 **MBTI 대표 오행** 배지 추가 표시 (내장 매핑 테이블)

#### 2.5 CTA 동작
- 4개 필드 모두 유효 → `#/loading` 전환
- 미입력 시 해당 필드 하단 inline 에러 메시지

#### 2.6 SEO·메타
- `<title>`: MBTI × 사주 오행 궁합 테스트 | 무료 궁합 점수
- `description`: 생년월일과 MBTI로 보는 하이브리드 궁합. 30초 만에 궁합 % 확인.

### SCR-01 — 로딩(분석 중) 화면

#### 2.1 목적
- **체류 시간·집중도** 극대화 구간 (애드센스 RPM 핵심)
- 사용자 기대감 형성 후 결과 화면으로 자연 전환

#### 2.2 레이아웃 구역

| 구역 ID | 명칭 | 내용 |
|---------|------|------|
| AD-LOAD | 로딩 광고 | 전면/디스플레이 광고 슬롯 (AdSense) |
| ANIM | 분석 애니메이션 | 오행 원형/별자리 회전 SVG 또는 CSS |
| MSG | 상태 메시지 | 순환 텍스트 (아래 2.3) |
| PRG | 진행 표시 | 0→100% 프로그레스 바 (2.5초) |

#### 2.3 순환 메시지 (2.5초, 3단계)

| 순서 | 문구 |
|------|------|
| 1 | 우주의 기운을 모으는 중... |
| 2 | 오행(목·화·토·금·수) 균형을 분석 중... |
| 3 | MBTI 성향과의 조화를 계산 중... |

#### 2.4 타이밍

| 이벤트 | 시점 |
|--------|------|
| 화면 진입 | 0ms |
| 광고 슬롯 렌더 | 100ms |
| 메시지 1→2 | 800ms |
| 메시지 2→3 | 1600ms |
| `#/result` 전환 | 2500ms |

#### 2.5 제약
- **뒤로가기**: 입력 화면으로 복귀 (입력값 sessionStorage 유지)
- **스킵 버튼 없음** (수익·UX 균형 — v1 기준)

### SCR-02 — 결과 화면

#### 2.1 목적
- 궁합 **% 점수**와 **한 줄 요약**으로 SNS 공유 유도
- 블로그 상세 해설 글로 **내부 트래픽** 유도

#### 2.2 레이아웃 구역 (위→아래)

| 구역 ID | 명칭 | 내용 |
|---------|------|------|
| SCORE | 궁합 점수 | 큰 숫자 % + 등급 이모지 (🔥💚💛) |
| PAIR | 커플 카드 | A(MBTI+오행) ↔ B(MBTI+오행) |
| SUMMARY | 한 줄 요약 | 40자 내외 |
| DETAIL | 상세 설명 | 오행 보완·MBTI 조합 2~3문단 (내장 템플릿) |
| AD-RESULT | 결과 광고 | 디스플레이 반응형 (공유 버튼 **바로 위**) |
| SHARE | 공유 액션 | 링크 복사, 결과 이미지 저장, (선택) SNS |
| BLOG-LINK | 블로그 유도 | "○○ 기운의 더 깊은 궁합이 궁금하다면?" → 블로그스팟 URL |
| RETRY | 다시 하기 | `#/` 이동, 폼 초기화 옵션 |

#### 2.3 결과 데이터 표시项

| 항목 | 예시 |
|------|------|
| `compatibilityScore` | 87 |
| `grade` | 🔥 찰떡 궁합 |
| `ohengBalance` | "금(金)이 화(火)를 단련해 주는 관계" |
| `mbtiSynergy` | "INTJ × ENFP — 서로 다른 세계를 탐험" |
| `shareText` | "우리 MBTI×오행 궁합 87% 🔥 INTJ(金) × ENFP(火) 테스트해보기 → [URL]" |

#### 2.4 등급 구간

| 점수 | 등급 | 이모지 |
|------|------|--------|
| 85~100 | 찰떡 궁합 | 🔥 |
| 70~84 | 좋은 궁합 | 💚 |
| 55~69 | 보통 궁합 | 💛 |
| 40~54 | 노력형 궁합 | 💪 |
| 0~39 | 주의 필요 | ⚠️ |
